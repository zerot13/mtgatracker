Release notes for 4.1.2 (and 4.1.1)

  • Duplicate notifications will now only appear once with an incrementing badge counter
  • Fixed an issue that caused an error message during application updates
  • Deck names can no longer cover the zoom buttons (thank you Alex!)
  • Fixed an issue where some users may be unable to authorize Inspector Ravi!) If you're still unable to authorize Inspector or cannot see matches in Inspector, please contact us on Discord!
  • Precon deck names will no longer show up as "?=?Loc/Decks/Precon/Precon_NPE_XY", and instead will show their in-game deck name.

Release notes for 4.0.0

  • Support for Guilds of Ravnica! (Open Beta HYYYYYPE)
  • We've redesigned MTGATracker's core model of trust to make Inspector logins easier
  • Notifications styling has been improved
  • Added our supporters to the Settings panel. We ❤ our supporters!

Release notes for 3.7.0

  • MTGATracker now tracks your collection. Watch out for an inspector update, soon!
  • MTGATracker can now export your MTGA collection (MTGGoldfish format)
  • MTGATracker now checks for updates more reliably, and shows if an update is processing in the settings window

Release notes for 3.6.1

  • MTGATracker now tracks how many mulligans you took in a game
  • MTGATracker will no longer allow you to zoom to impossible zoom levels
  • Electron version updated

Release notes for 3.6.0

  • You can now authorize MTGATracker to upload data to your inspector account, in the "Inspector" tab. Note: This will be required after 9/10/18! Read more on the MTGATracker blog.
  • MTGATracker now tracks your draft history (thanks isharacomix for the PR!) Look for an inspector update in the near future to see your draft history! (requires authorizing inspector)
  • MTGATracker now tracks changes to your collection (vaultProgress, Gold, and Gems--requires authorizing inspector)

Release notes for 3.5.8

  • Fixed a logparsing issue introduced in the 8/16 MTGA patch

Release notes for 3.5.7

  • Support for Core Set 19 cards

Release notes for 3.5.5

  • Cards that were played by you are now reported to Inspector
  • Implemented "Chess Timers" in the tracker
  • The hide timeout can now be configured in Settings
  • Hide functionality can now be inverted (not recommended unless you know you want it)
  • Improved logging between python backend and electron frontend

Release notes for 3.5.0

  • Player who is "on the play" for each game now reported to inspector
  • Rank change now reported to inspector
  • Opponent's rank (before game) now reported to inspector
  • Length of games (elapsed time & turn count) will now be reported to Inspector
    • (Keep an eye out for an inspector update soon!)
  • When using streamer mode, window will no longer resize dynamically
  • Tracker will now remember previous zoom level on launch

Release notes for 3.4.2

  • Implemented a new default theme; read more about it here!
  • Fixed an issue that could in rare occasions cause the tracker to get stuck / games to fail to upload to inspector
  • MTGATracker will now be scrollable when the window is too large to fit on the screen

Release notes for 3.4.0

  • Added new Draft Collection View feature. When drafting, MTGATracker will show you how many copies you own of each pick!
  • Added Terms of Services agreeement

Release notes for 3.2.2

  • Fixed an issue in 3.2.1 & under where the 2nd+ game in competitive draft would upload multiple times and/or cause upload error messages
  • Added deck list sorting options to General tab of settings; sorting method "Emerald" implemented
  • When using multiple arts for the same card in the same deck, that card will now be condensed into a single row

Release notes for 3.2.0

  • Added support for KLD, AER, W17 sets
  • Refactored json-blob parser task post-KLD patch

Release notes for 3.1.0

  • Added Win / Loss counter (disable or reset in General tab)
  • Added auto-hide on right click (disable or reset in General tab)
  • Tracker will now open settings to about page on first launch after an update (so you can see this changelog!)
  • Tracker will now attempt to upload at end of game slightly more aggressively, to help prevent lost inspector data
